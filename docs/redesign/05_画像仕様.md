# 05. 画像仕様書 — 生成プロンプトと差し替え運用

> 対象: `project/assets/` の写真スロット P01〜P35。
> HTML側の `data-photo="Pxx"` 属性は差し替えの座標なので**絶対に消さない**。
> 生成→検品→差し替えの運用ルールは §4。オリジナル写真への最終差し替えは §5。

## 1. 共通プロンプト部品（全スロットに前置）

**BASE（英語・そのままコピペ用）:**
```
Photorealistic professional photograph, bright modern indoor swimming pool in Japan,
natural daylight, clean blue water with lane ropes, soft warm color grading,
slightly muted saturation, documentary sports-lifestyle style.
All people are Japanese adults (late 20s to 50s). No children anywhere.
Absolutely no text, no letters, no logos, no watermarks, no graphic overlays in the image.
```

**NEGATIVE（対応モデルでは negative prompt に）:**
```
children, kids, text, typography, letters, watermark, logo, cartoon, illustration,
oversaturated colors, distorted hands, extra fingers, deformed face
```

**フィン種別の語彙（プロンプト内で使い分け）:**
- ビーフィン: `a pair of short training swim fins (bi-fins)`
- モノフィン: `a single wide-blade monofin (both feet in one large fin, mermaid-style competitive swimming fin)`

## 3. 生成時の技術指定（codex CLI ほか画像モデル共通）

- 出力: PNG、表示サイズの約2倍で生成 → 後で最適化（06実装計画でWebP化）
- 同一人物が複数スロットに出る場合（コーチ: P01/P12/P22/P34/P35）は**同一シードや参照画像機能で顔を揃える**か、顔がはっきり写らない構図を選ぶ
- 生成後チェックリスト:
  - [ ] 文字・ロゴが写り込んでいない
  - [ ] 子どもが写っていない
  - [ ] フィン種別が指定どおり（ビ/モ）・形状が現実的
  - [ ] 手指・顔の破綻なし
  - [ ] トーンがP01（実写真）と並べて違和感なし

## 4. 差し替え運用（AI生成版）

1. 生成した画像を `project/assets/` の**同名ファイルに上書き**（アスペクト比を守れば레イアウト無傷）
2. `python3 -m http.server` でプレビューし、該当セクションを375px/1366pxで目視
3. コミットメッセージに `photo: P05,P14 を大人被写体に差し替え` のようにP番号を記録

## 5. オリジナル写真への最終差し替え（撮影後）

- 撮影リストはこの台帳の「個別プロンプト」をそのまま**撮影指示書**として使う（構図・被写体・雰囲気が言語化済み）
- 優先撮影: P35（コーチ本人）→ P01（FV）→ P17〜P19（声・実際の受講者）→ レッスン風景（P08〜P13は1回の練習で一括撮影可能）
- ★本人顔スロット: P01・P12・P16・P22・P34・P35（旧ガイドのP04/P24は廃止・変更）
- 撮影時の注意: 会場の撮影許可・参加者の肖像権同意書（テンプレは要準備）
