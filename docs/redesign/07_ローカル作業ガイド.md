> クラウドセッション（claude.ai/code）からのハンドオフ資料。ローカルのClaude Codeで
> このリポジトリをそのまま進められるように、体制・手順・コピペ用プロンプトをまとめる。

## 1. セットアップ（初回のみ）

```bash
cd /Users/yoshi/projects/Fin-D_SS_skill_up_class/LP
git pull
claude
```

- **起動場所は `LP/` 直下を推奨**。Claude Codeは親フォルダのCLAUDE.md（Fin-D_SS_skill_up_class/ に作り込まれた設定）も自動で読み込むため、既存のトークン効率化がそのまま効く
- このリポジトリ自身の `CLAUDE.md` に厳守ルール・経緯・確定事実が全て記録済み

## 2. モデル振り分け体制（トークン効率）

リポジトリに**エージェント定義を同梱済み**（`.claude/agents/`）。ローカルでもクラウドでも同じ体制で動く:

| 役割 | エージェント | モデル | 使い方 |
|---|---|---|---|
| 実装 | `lp-builder` | **sonnet** | 「lp-builderにPhase 1の◯◯を実装させて」 |
| レビュー | `lp-reviewer` | **opus** | 実装後に「lp-reviewerで監査して」 |
| 設計判断 | メインセッション | 任意 | 設計書の解釈・変更判断のみ |

- メインループも節約するなら起動後に `/model sonnet`（設計判断が必要な場面だけ `/model opus` 等へ）
- 原則: **メインセッションは指揮とレビュー結果の判断だけ**を行い、ファイル編集・検証はlp-builderに投げる

## 3. 作業の流れ（1フェーズ = 1サイクル）

```
git pull（必ず最初に）
→ lp-builder に実装させる
→ lp-reviewer に監査させる
→ 指摘があれば lp-builder に修正させる
→ コミット & push（フェーズ単位）
```

⚠ **クラウドセッションと同時に編集しない**こと。どちらかで作業→push→もう片方でpull、を徹底（コンフリクト防止）。

## 4. フェーズ実行用プロンプト（コピペ用）

### Phase 1: ビジュアルの大人化
```
lp-builderエージェントで docs/redesign/06_実装計画.md の Phase 1 を実装して。
仕様は 04_デザインシステム.md（カラートークン差し替え・見出し装飾の統一・
装飾SVG削減・角丸12px統一・影統一・コントラスト修正）。
デスクトップの構造は壊さず「表皮の置き換え」に徹すること。
完了したら lp-reviewer で監査し、指摘を修正してからコミット・push して。
```

### Phase 2: 構成改修（コーチ紹介新設・Members圧縮）
```
lp-builderエージェントで docs/redesign/06_実装計画.md の Phase 2 を実装して。
コーチ紹介セクション(#coach)の仕様は 03_構成設計.md §7、
素材（実名・肩書き候補・指導哲学の実文）は 00_事実データシート.md にある。
コーチ写真は project/assets/coach.png（無ければプレースホルダーで組んでおく）。
Members圧縮・アプリモック非表示化・導入圧縮も同Phaseの範囲。
完了したら lp-reviewer で監査→修正→コミット・push。
```

### Phase 3: 画像差し替え
```
docs/redesign/05_画像仕様.md に従って画像を差し替える。
★★★の8スロット（P05/P14/P18/P21/P22/P23/P28/P33）が最優先。
生成は codex CLI で05のプロンプトをそのまま使用（BASE+個別+NEGATIVE）。
生成後は同名で project/assets/ に上書き→375/1366pxで目視→
05 §3のチェックリスト（文字焼き込み・子ども・フィン種別・破綻）を通してからコミット。
```

### Phase 4: CV基盤（計測・SEO・法的表記）
```
lp-builderエージェントで docs/redesign/06_実装計画.md の Phase 4 を実装して。
フォームは接続済み（https://fin-d-skillup-form.vercel.app/f/skillup-july-2026。
月替わりでslug更新）。残りは GA4イベント・JSON-LD・OG画像・WebP化・
```

### 実声化（アンケート受領後）
```

```
